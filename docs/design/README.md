# Design Notes Index

This folder contains supporting design, quality, refactoring, and assignment-defence notes.

These files do not replace the repository's canonical documents. `AGENTS.md` is the single
operational instruction source for AI coding agents. The root documents remain authoritative for
architecture, rules, domain modelling, data, and rationale.

## Files

- `architecture-quality-checklist.md` - pre-submission and Sprint 3 architecture checklist.
- `object-oriented-principles.md` - focused OO principles applied to the implementation.
- `patterns-and-rationale.md` - concise pattern rationale and rejected-pattern notes.
- `refactoring-antipatterns-guide.md` - evidence structure and anti-pattern/refactoring guidance.
- `antipatterns-and-refactoring.md` - pointer to the canonical deliverable at the repository root
  (`antipatterns_and_refactoring.md`).
- `quality-scenarios-and-verification.md` - quality attributes, scenarios, and verification checks.
- `assignment-defence-guide.md` - report/viva talking points grounded in implementation evidence.

## Use Policy

Use these files as supporting material when designing, reviewing, refactoring, or writing the
assignment report. If a supporting note conflicts with `AGENTS.md`, `marking_rubric.md`,
`game_rules.md`, `domain_model.md`, `destination_cards.md`, `london_board_layout.md`, or
`architecture_guide.md`, follow the higher-priority source listed in `AGENTS.md`.
